# ReactionProfileHunter S0 机理分类模块重构与升级方案 (修订版)

## 1. 引言
S0机理分类模块旨在为ReactionProfileHunter流水线注入机理感知能力，通过对反应物、产物及试剂的化学信息学解析，生成结构化的机理表达式（MechanismExpression），从而驱动后续流水线的精确执行。在综合最新的架构审计意见后，本案取消了极具野心但目前不具备数据支撑的ML连续描述符，回归实用主义：**通过严谨的离散图模型（DAG）、预定义的 YAML 专家系统以及严格限定 S0 的“纯拓扑/2D”的职责界限**，来指导高度自动化的量子化学流水线。

---

## 2. 核心数据模型重构
### 2.1 从线性模型向有向无环图 (DAG) 跃迁的深度解析
废弃僵化的线性 `step1/step2` 字段。为了原生表达多步骤、竞争路径和立体异构网络，引入全新的 `MechanismGraph`。

**数据结构与算法设计**：
- **底层依赖**：采用 `networkx` 构建 `DiGraph`（有向图），并封装进 Pydantic 模型（如 `MechanismGraphModel`）以实现强类型校验和 JSON 序列化。
- **节点 (Node - 驻点)**：代表势能面驻点（反应物、多分子络合物、中间体、产物）。
  - `node_id`: 唯一标识 (如 `N_reactant_1`, `N_int_dipole`)
  - `smiles`: 标准化的二维结构标签（**注意：S0 仅负责输出中间体的 2D SMILES 作为拓扑守恒校验的锚点，绝对不进行任何 3D 坐标生成**）
  - `charge` / `multiplicity`: 物理化学性质
  - `state_type`: 类型枚举 (`reactant`, `intermediate`, `product` 等)
- **边 (Edge - 基元反应/过渡态)**：代表节点间的基本物理演化路径。
  - `forming_bonds` / `breaking_bonds`: 成键与断键原子索引集（供 S2 模块直接消费）
  - `ts_type`: 倾向 (`concerted`, `stepwise_first_C_C`, 等)
- **图的动态生成与S2的对接**：S0 根据选定的 YAML 模板，用 `networkx.compose` 或节点逐步添加法生成完整的反应 DAG。S2 (`RetroScanner`) 在执行时，将逆向遍历（从 Product 回溯至 Reactant）该 DAG 的 Edge。S2 依据边上定义的 `forming_bonds/breaking_bonds` 进行 **产物结构的坐标反向拉伸（Retro-stretching）** 以自动寻找中间体或过渡态的 3D 构型。S2 验证找到的 3D 构型是否符合该对应 Node 的 2D SMILES 拓扑信息。

## 3. 化学信息学引擎深度升级

### 3.1 引入 RDChiral 替代原有 RDKit 反应引擎
RDKit 的原生 `ReactionFromSmarts` 在手性中心（引入/破坏/翻转）和双键 E/Z 构型保持上存在致命缺陷。引入 **RDChiral** 严格执行立体化学拓扑守恒，所有的中间体 2D 模板生成要求：
- 每条 `rxn_smarts` 通过 RDChiral 验证。
- 确保从前体 SMILES 生成中间体 SMILES 时，所有未参与反应的手性环境百分百守恒。

### 3.2 明确的 3D 构型生成责任链
**重要修正**：S0 模块**严禁**为偶极子中间体尝试生成 3D 坐标（如调用 ETKDG 或 xTB）。通过 1D SMILES 平白无故生成中间体的 3D 构象是极不靠谱的化学行为。
- **S1 (AnchorPhase) 的职责界限**：S1 仅对稳定态分子（反应底物、已知试剂、最终稳态产物）进行基于 SMILES 的 3D 构象搜索与对齐。
- **S2/S3 的职责界限**：所有高能中间体（如 1,3-偶极子）的 3D 构型，**完全由 S2 通过拉伸已知产物的 3D 拓扑结构倒退生成**。S0 在此处仅提供该中间体的 `[预期 2D SMILES 图结构]` 作为 S2 验证生成的 3D 中间体拓扑是否正确的**校验标准**。

### 3.3 SMARTS 编译与性能优化
预编译所有 SMARTS 至 Mol 对象，并对匹配逻辑的执行顺序实施动态树剪枝：
- **AND 节点**: 最严格的约束基元前置，执行尽早剪枝。

## 4. 基于实用主义的机理匹配与人工标注机制

针对小数据集和先期开发，**取消复杂易过拟合的连续特征打分 (CGR/DRFP)、神经网络 (GNN) 及高度模棱两可的发散态概率输出系统。**

### 4.1 角色分配算法 (Role Allocator)
摈弃预先标注前体功能的假设模式，改为适应真实的无序“反应池”：
- 输入：无序的反应物和试剂列表 (SMILES list)。
- 求解：针对每个机理模板定义的“角色”（如亲双烯体、呋喃衍生物等），利用 **最大权重二分图匹配算法 (KM 算法)** 进行约束求解匹配，确定反应位点和分子身份。

### 4.2 专家系统级打分机制 (硬约束 + 软证据)
- **硬约束 (Hard Constraints)**：通过 RDKit 过滤绝对不可行的模板（例如缺乏形成偶极子所需的氧化剂/离去基团）。
- **人工标签介入 (Manual Overrides)**：鉴于极高精度的自动化分类在目前百量级反应数据下容易出现伪规律，在输入端开放强制指定机理模板的接口（例如在输入配置中写入 `force_mechanism: [4+3]_DMDO_TypeA`）。当自动化分类无法得出唯一解时，信任人工配置标签而非进行过度泛化。

## 5. 区域选择性与立体化学 (dr) 的平铺式预测

**重要修正**：不要在 S0 阶段试图基于空间位阻（如三烷基硅基）或电子效应来提前“剪枝”掉劣势构型（如预测 `exo` 极不利而直接移除），因为规则极易出现漏洞或无法量化具体能垒差异。
- **平行拓展策略**：S0 的角色是在 DAG 中“展开”所有合理的立体异构体（如针对 `diastereomeric ratio, dr` 问题）。
- 当模板检测到潜手性中心或面选择性发生时，S0 会生成两条**对等的、平行的多叉路径**（如 `Path_Endo` 与 `Path_Exo`），并将其全部写入 MechanismGraph。
- 后续 S2 和 S3 会独立平行地下发这两条路径的实际 DFT 搜索计算。最后依靠 S4 提取的精确量子化学能量差 ($\Delta\Delta G^\ddagger$) 来判定实际的 dr 比例。
- S0 仅做“选项枚举”而非“选项决定”。

## 6. 与下游流水线的自适应闭环集成

### 6.1 S1: 状态感知型力场自适应
根据 S0 输出的稳态分子 `charge` 和 `multiplicity` 智能切换构象引擎的基础力场。对于中性闭壳分子部署 MMFF，对抗常规的开壳分子主动切为 `GFN2-xTB` 以免力场崩溃（仅限 S1 应负责处理的稳态前体与产物）。

### 6.2 S2/S3: 原生映射断键与成键约束
S0 生成完整的边属性 `forming_bonds` / `breaking_bonds`。S2 接收此参数后，直接将其作为扫描的距离约束矩阵，实现基于化学拓扑机理“照本宣科”式的精确拉伸扫描操作。

### 6.3 失败回调扩展 (未来支持)
为后续（拥有足够多替代策略时）预留 Orchestrator 回调 API，当 S3 断键失败或 IRC 脱轨时，能追踪到 DAG 中替代的分支机制重新尝试。

## 7. 实施路线图建议 (Implementation Roadmap)

1. **Phase 1: 图论数据模型与基础框架 (2周)**
   实现基于 `networkx` 和 `Pydantic` 的 `MechanismGraph` 数据容器，完成 JSON 序列化。封装 RDChiral 代替原生反应引擎，建立稳态分子 2D->2D 的生成校验。
2. **Phase 2: 匹配引擎与专家模板迁移 (3周)**
   部署 KM 分配算法处理无序底物；构建基于硬约束的匹配过滤系统，开发手动 YAML 干预接口。对现有的近 20 种 [4+3] 与 [5+2] 机理重写专家模板。
3. **Phase 3: S2/S3 拉伸协同适配 (2周)**
   调整 S2 The RetroScanner 流水线，使其摒弃过去的“盲目”扫描，完全根据 S0 MechanismGraph 的边指定的 `forming_bonds` 与端点 2D 拓扑结构来定向生成中间体 3D 坐标。
4. **Phase 4: dr 的平行枚举集成测试 (2周)**
   选取文献中存在明确 `endo/exo` 竞争的体系，测试 S0 是否能正确输出包含双规并行的 DAG，并确保 S2/S3 能同时完成两套 TS 优化并得出对应的能量基准。
